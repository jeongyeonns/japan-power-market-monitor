## 조사 대상과 읽기 방식

- 원본 파일: `data/eprx/202607_1-0_prompt.csv`
- 파일 형식: CSV, 쉼표(`,`) 구분
- 파일 크기: 1,212,248 bytes
- SHA-256: `a9b8446a2e4e1068d4c0530ec62b14b8edb0387b45fd8f378f7716d34299c915`
- 선택 인코딩: **cp932**
- 헤더 구조: 1행 `H` 파일 설명, 2행 `P` 상태·대상월·상품·블록 수,
  3행 `TT` 지역 이름, 4행부터 시간대별 13개 지표 반복
- 데이터 범위: 2026-07-01~2026-07-21
- 파일 상태: `速報値`(속보치)
- 상품 표기: `一次調整力`

인코딩은 요청된 순서인 cp932, shift_jis, utf-8-sig, utf-8로 원본 바이트를
엄격 디코딩했습니다. cp932와 shift_jis는 일본어 열과 첫 10개 데이터 행이
정상이고 `�`가 0개였습니다. 우선순위가 높은 cp932를 선택했습니다.
utf-8-sig와 utf-8은 바이트 위치 2의 `0x93`에서 실패했습니다. 이 결과는 앱의
파일 처리 로그 `encoding`과 `encoding_attempts`에도 표시됩니다.

## 날짜·시간대와 단위

- 거래 ID 형식: `YYYYMMDD` + `B` + 2자리 블록 번호
- 예: `20260701B01`
- `delivery_date`: 앞 8자리, `YYYYMMDD`
- `period_no`: `B01`~`B48`
- `period_start`: B01=00:00, B02=00:30, …, B48=23:30
- 가격 단위: `円/kW・30分`
- 물량 단위: `MW`
- 시간 기준 표기가 파일 안에 있는지: **확인 필요**. 앱은 프로젝트 원칙에 따라
  Asia/Tokyo 달력으로 취급합니다.

## 확인 필요 및 주의사항

- `速報値`가 이후 어떤 파일에서 `確報値`로 표기되는지는 추가 파일 확인 필요
- Excel 시트 구조는 실제 Excel 파일이 없어 확인 필요
- 다른 대상월이나 다른 상품 파일도 동일 구조인지 추가 파일 확인 필요
- 원본 CSV를 Excel에서 열었을 때 보이는 깨진 문자열은 사용하지 않습니다.
- 원본 파일은 수정하거나 덮어쓰지 않으며 Python에서 cp932로 직접 읽습니다.
- EPRX 자동 다운로드, 웹 스크레이핑, 임의 단위 변환은 구현하지 않습니다.
