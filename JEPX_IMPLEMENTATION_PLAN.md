# JEPX 구현 계획

1단계의 실제 CSV 구조 분석과 정규화 로더 연결을 완료했습니다. 가격 프로파일과
ESS 스프레드는 아직 계산하지 않습니다. 새 파일 형식은 실제 원본을 확인한 뒤에만
지원하며 열 이름, 파일 형식 또는 가격 구조를 추측하지 않습니다.

## 단계별 계획

1. 실제 JEPX Day-Ahead 원본 파일 수동 연결 및 구조 분석 — 완료
2. 날짜, 30분 시간대, 시스템가격, 지역가격 표준화
3. 데이터 품질 검증 및 지역명 표준화
4. ESS 스프레드 계산 정의 확정
   - 30분
   - 1시간
   - 2시간
   - 4시간
5. 일별·주별·월별·연도별 집계
6. 지역별 가격 및 스프레드 비교 화면
7. 전주 및 전월 대비 변화
8. 규칙 기반 시장 요약
9. NEM 앱과 같은 AI 분석 기능
   - 계산 결과 기반 해석
   - 핵심 수치 표시
   - 그래프 설명
   - 후속 질문
   - 필요 시 선택적 웹 검색

## 구현 원칙

- AI가 원시 가격이나 스프레드를 직접 계산하지 않습니다.
- 검증된 Python 계산 결과만 AI에 전달합니다.
- 실제 파일을 확인하기 전에는 열 이름을 추측하지 않습니다.
- EPRX와 JEPX의 파서 및 분석 모듈을 분리합니다.
- 사용자 화면은 한국어로 표시합니다.
- 내부 표준 컬럼명은 영어로 유지합니다.
- JEPX의 필터, 캐시 및 session state에는 `jepx_` 접두어를 사용합니다.
